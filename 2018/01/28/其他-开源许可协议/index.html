其他-开源许可协议

2018-01-28 • 分类: 技术栈 • 标签: others

常用的开源协议有六种：GPL, LGPL, BSD, Apache, Mozilla, MIT

常用协议介绍

BSD（original BSC license, FreeBSD license, Original BSD license）

“为所欲为”：可自由使用，修改源代码，修改后可作为私有的或者开源的发布。

发布BSD协议的代码或者基于BSD修改开发时，需要：

  • 修改后包含的源代码仍然要有BSD协议
  • 如果再发布的是二进制库或软件，版权声明中要说明包含原来的BSD协议
  • 不可以用开源代码的名字、作者或机构信息进行推广

Apache Licence 2.0(Apache License, Version 2.0 、Apache License, Version 1.1 、Apache License, Version 1.0)

非盈利开源组织Apache采用的协议，鼓励代码共享，允许修改与再发布为商业或开源的

  • 向再发布的代码的用户提供一份Apache Licence
  • 如果对于代码进行了修改，需要在被修改的文件中进行说明
    • 怎么说明？
  • 延伸代码中要带有原来代码的协议，商标，专利声明以及其他作者规定的要包含的说明
  • Notice文件（如有）需要带有Apache Licence，可以在其中增加自己的许可，不可以修改AL

GPL(GNU General Public License)

GPL不允许修改后的、衍生的代码作为闭源商用软件，但是有传染性

  • “传染性”：只要一个软件中使用了GPL协议的代码，这个软件也必须GPL-开源、免费
  • 不适合商业使用

LGPL(GNU Lesser General Public License)

  • 允许商业软件通过类库引用（link）的方式，使用LGPL类库，但是不需要开源商业软件代码
    • 类库应用？
  • 如果是修改、衍生的LGPL代码，则必须是LGPL协议
  • 适合作为第三方库引用

MIT (The MIT License)

  • 类似于BSD，作者只需要保留版权，没有任何其他限制

MPL (Mozilla Public License)

  • 免费重发布，免费修改
  • 修改后代码版权归软件的发起者
